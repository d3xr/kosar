## Контекст

Проект уже содержит firmware, static site, docs и Obsidian-контекст. Нужен единый агентный слой, чтобы разные чаты и агенты не теряли правила работы, историю решений и карту документации.

## Решение

Адаптировать `2030ai/2030ai-project-template` без переезда существующих рабочих папок:

- добавить `agent_docs/`;
- добавить `AGENTS.md` как проектный контракт;
- добавить `.editorconfig`, `.markdownlint.json`, `.env.example`, `.cursorignore`, `.vscode/settings.json`;
- добавить skill mirror folders;
- не добавлять `agent_docs/setup-checklist.md`, так как проект уже живой и checklist пройден в рамках миграции.

## Обоснование

Такой вариант сохраняет текущие пути firmware/dashboard/docs, но даёт агентам шаблонную карту проекта, ADR и development history.

## Последствия

Плюсы:

- новым агентам проще входить в проект;
- решения фиксируются атомарно;
- снижается риск “разъезда” сайта, прошивки и документации.

Минусы:

- появляется больше документации, которую надо поддерживать;
- markdownlint workflow может потребовать правки старых markdown-файлов при будущих изменениях.
