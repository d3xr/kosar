# История разработки

Журнал итераций проекта. Одна нетривиальная итерация — один файл.

## Правила

- Имя файла: `YYYY-MM-DD-HHMM-short-title.md`.
- Время локальное для проекта/сессии.
- Не вести ручную сквозную нумерацию.
- Не ротировать и не архивировать старые записи.
- Не обновлять этот README при каждой новой записи.

## Когда создавать запись

Создавать для новых функций, архитектурных решений, существенных фиксов, миграций, публикаций и крупных документных изменений.

Не создавать для мелких формулировок, форматирования и очевидных одношаговых изменений.
