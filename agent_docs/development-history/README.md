# История разработки

Журнал итераций проекта. Одна нетривиальная итерация — один файл.

## Правила

- Имя файла: `YYYY-MM-DD-HHMM-short-title.md`.
- Время указывать локальное для проекта/сессии.
- Не вести ручную сквозную нумерацию.
- Не ротировать и не архивировать старые записи: дата в имени файла уже задаёт порядок.
- Не редактировать чужие старые записи без явной необходимости. Если нужно исправить контекст, создать новую запись и сослаться на старую.
- Не обновлять этот README при каждой новой записи.
- Глубину просмотра истории агент определяет сам по задаче: учитывать риск, затронутые файлы, связанные ADR и результаты поиска по ключевым словам.

## Когда создавать запись

Создавать для нетривиальных изменений: новая функциональность, изменение правил проекта, архитектурное решение, исправление существенного бага, миграция, крупная реструктуризация.

Не создавать для мелких правок формулировок, форматирования и очевидных одношаговых изменений, если они не меняют правила, поведение или структуру проекта.
