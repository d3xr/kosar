# Архивация и временные файлы

## Архивация

Архивировать только при замене важных файлов, когда старый вариант нужен как исторический артефакт.

- Папка: `archive/`.
- Имя: `[YYYY-MM-DD_HH-MM]_original_name.ext`.
- В начало архивного файла добавить комментарий с причиной, датой и новой версией.

Не архивировать:

- git-файлы;
- секреты;
- временные файлы;
- generated build artifacts;
- записи из `agent_docs/development-history/` и `agent_docs/adr/`.

## Временные файлы

- Складывать в `temp/`.
- Чистить после задачи.
- Если временный формат повторяется — добавить в `.gitignore`.
