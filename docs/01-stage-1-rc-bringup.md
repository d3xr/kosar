# Stage 1 RC Bring-Up

Цель этапа: получить медленную, управляемую и останавливаемую RC-платформу на родной гироскутерной ходовой. Нож Greenworks не подключается.

## Gate 0. До прошивки

- Сфотографировать плату целиком, MCU, SWD-площадки, разъемы sideboard, фазные провода, Hall-провода, батарею, MOSFET, кнопку питания.
- Подтвердить, что это Gen1: одна центральная mainboard управляет двумя моторами.
- Проверить штатную работу гироскутера до разборки: оба колеса крутятся, батарея заряжается, механического хруста нет.
- До прозвонки ничего не резать и не перепаивать.

## Gate 1. Toolchain

На текущей машине не найдены `openocd`, `st-flash`, `platformio/pio`, `arm-none-eabi-gcc`.

Минимальная установка на macOS:

```bash
brew install openocd stlink
python3 -m pip install --user platformio
```

Проверка:

```bash
openocd --version
st-flash --version
pio --version
```

## Gate 2. Получить firmware

Рабочую копию upstream держать вне git history этого repo:

```bash
mkdir -p firmware/upstream
git clone https://github.com/EFeru/hoverboard-firmware-hack-FOC.git firmware/upstream/hoverboard-firmware-hack-FOC
cd firmware/upstream/hoverboard-firmware-hack-FOC
git log -1 --format='%H %cs %s'
```

## Gate 3. Первый конфиг

Для прямого PPM/PWM:

```bash
pio run -e VARIANT_PPM
# или
pio run -e VARIANT_PWM
```

Для ESP32 bridge:

```bash
pio run -e VARIANT_USART
```

Начальные ограничения для bench-теста:

- `BAT_CELLS 10`.
- `CTRL_TYP_SEL FOC_CTRL`.
- `CTRL_MOD_REQ VLT_MODE`.
- `FIELD_WEAK_ENA 0`.
- `TANK_STEERING` выключен.
- `I_MOT_MAX` начать консервативно, затем повышать по нагреву и тяге.
- `N_MOT_MAX` начать низко; дефолтные 1000 rpm для 10" колес слишком быстро для первой тележки.

## Gate 4. Unlock / flash

ST-Link подключать только к `GND / SWDIO / SWCLK`. `3.3V` от ST-Link не подключать. Плата питается отдельно.

Команда unlock из EFeru Makefile:

```bash
openocd -f interface/stlink-v2.cfg -f target/stm32f1x.cfg -c init -c "reset halt" -c "stm32f1x unlock 0"
```

Важно: unlock стирает штатную прошивку. Перед этим зафиксировать фото платы и факт, что возврата к стоку может не быть.

При прошивке удерживать кнопку питания или временно зафиксировать power latch, иначе плата может выключиться в середине операции.

## Gate 5. Smoke test

- Первый запуск после прошивки: CC-блок 36-42V с ограничением 1-2A, если есть подходящий источник.
- Колеса вывешены.
- Подавать минимальные команды.
- Проверить, что нет мгновенного current limit, дыма, перегрева MOSFET, рывков.
- Если мотор дергается или гудит: остановиться, проверить фазы/Hall, не повышать газ.

## Gate 6. RC test

- Общая земля между приемником/ESP32 и платой обязательна.
- Не питать приемник от sideboard cable до проверки мультиметром: на красном проводе может быть 12/15V.
- Проверить центры каналов и deadband.
- Проверить направления: вперед, назад, поворот влево/вправо.
- Проверить failsafe тремя сценариями: выключить пульт, потеря питания приемника, выдернутый signal wire.

## Gate 7. Greenworks mount

Только после стабильного bench-теста:

- Нож физически отключен.
- Батарея ножа снята или цепь ножа разомкнута.
- Силовые крепления колес из металла.
- Ось защищена от проворота.
- Кабели защищены от травы, влаги и движущихся частей.
- E-stop доступен снаружи рукой.

## Первый тест на земле

- Ровная площадка.
- Минимальная скорость.
- Оператор рядом.
- Запрещены pivot-turn и резкий реверс.
- После 1-2 минут: остановка, проверка температуры платы, моторов, проводов, разъемов.
