# Косарь — landing design code

## Идея

Лендинг — не красивый отчёт, а уличный тех-лог: что построили, как едет, где живёт контроль и где код/доки.

## Базовый тон

- Язык: русский, прямой, короткий, по-гаражному.
- Копирайт без “статейных” предупреждений и без лексики “без ножа/безопасность/лигал”.
- Никаких публичных обещаний пуска на моторы через интернет.
- Акцент на инженерный контур: ESP32 локально рулит, сайт — телеметрия и история.

## Архитектурная рамка страницы

- Public surface = read-only.
- Control surface = ESP32 + локальная прошивка + hoverboard UART.
- Dashboard = наблюдение каналов, статусы, скорость/руль, история логов.

## Контентная матрица

- Hero: 1 сильная фраза + 2 строки контекста.
- Story:
  - Зачем строим.
  - Из чего сделано.
  - Граница системы (что сайт делает, а что не делает).
- Cockpit:
  - статус-чипы;
  - 4 метрики;
  - barras каналов 8-16;
  - терминал событий.
- Build:
  - отвечать на вопрос “что нужно, чтобы собрать робота-газонокосилку”.
  - перечислять доноров и компоненты: газонокосилка, ховерборд, ESP32, ELRS, питание, крепёж, программатор.
  - не писать абстрактные этапы типа “стенд, затем корпус, затем улица”, если они не помогают повторить сборку.
- Concept:
  - объяснять зачем проект существует, какая эстетика и почему этот путь интереснее готовой пластиковой коробки.
  - не использовать фразы-пустышки вида “не учебный чертёж, а рабочий контур”.
- Repo/Credits:
  - прямые ссылки и короткий список артефактов.

## Визуальный код

Полные токены: `docs/design-system.md`.

### Цвета

- фон `#14100e`
- каркас `#2a2320`
- линия `#4a4038`
- текст `#e8dcc8`
- акцент `#e8801a`
- тревога `#ffb641`
- ошибка `#ff4b33`

### Типографика

- заголовки: `monospace`, контрастные, малые радиусы и плотная сетка.
- body: читаемая, но не “плоская”, второстепенный акцент через размер и цвет.

### Layout

- `hero` с крупным фоном и явным overlay.
- `grid`/`panel` блоки с чёткой геометрией.
- минимальный скругления (до 4px).
- на мобильном: единичная колонка, без сжатия текста, без “перепрыгивания” элементов.

## Что запрещено

- “without blade / безопасный режим / не включать без ...”
- “платформенный” пафос из другого домена.
- “нейро-слоп” формулы: “не X, а Y”, если они не несут конкретики.
- публичные инструкции по remote-контролю моторов.
- термины `safety`/`failsafe` как маркетинг; в технике — только по месту.
